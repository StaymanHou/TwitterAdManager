TwitterSession¶

class Lib.TwitterSession.TwitterSession(account)¶

A Twitter Session object. Contains a Lib.TwitterAccount.TwitterAccount object and a requests.session object. It also maintains auth_token and root_url after login. is_login holds the status of the session whether it’s login or not. You can use it just as a requests.session.

>>> ts = TwitterSession(twitterAccount)
>>> url = ...
>>> r = ts.get(url)
>>> payload = {...}
>>> r = ts.post(url, payload)

It will guarantee to login the twitter account before get() and post(). So you do not have to call Lib.TwitterSession.TwitterSession.login(). You can get the auth_token and the root_url by calling Lib.TwitterSession.TwitterSession.get_auth_token() and Lib.TwitterSession.TwitterSession.get_root_url(). They will alse guarantee the login. Do not try to access auth_token and root_url directly. It will be a empty string before login.

commit_login()¶

Return 0 if success, otherwise return none zero. Commit login and perform the behavior.

get(url, **kwargs)¶

Commit a get requests with self.session. It will guarantee to login first.

get_auth_token()¶

Return the self.auth_token. It will guarantee to login first.

get_root_url()¶

Return the self.root_url. It will guarantee to login first.

login()¶

Reset the self.session and use the session to login Infinite loop with an increasing retry waiting time until successful login. It calls Lib.TwitterSession.TwitterSession.commit_login() to commit login. Please invoke this method at the begining of other method to make sure the login status.

post(url, **kwargs)¶

Commit a post requests with self.session. It will guarantee to login first.
